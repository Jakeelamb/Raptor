# Genome Assembly Benchmarking

This directory contains the reproducible benchmark workflow for positioning Raptor against established genome assemblers.

## Goals

- compare Raptor against baseline assemblers using fixed commands
- measure runtime, peak memory, and assembly continuity
- keep scripts and methodology in git while keeping generated artifacts out of git
- produce benchmark summaries that can be cited in docs, issues, and release notes

## Layout

- `setup_environment.sh`: installs benchmark dependencies
- `environment.yml`: pinned conda environment for benchmark tooling
- `download_data.sh`: downloads public datasets or generates simulated data
- `quick_benchmark.sh`: fast smoke benchmark on synthetic data
- `run_benchmark.sh`: full benchmark run for one dataset
- `summarize_results.py`: aggregates multiple runs into CSV and Markdown tables

Generated paths:

- `data/`: downloaded or simulated inputs
- `results/`: per-run outputs, logs, and reports

These generated directories are intentionally ignored by git.

## Recommended Comparators

Current workflow includes SPAdes by default. For a stronger publication story, extend the same harness to include:

- MEGAHIT
- ABySS
- Velvet

Do not publish cherry-picked results. Use the same datasets, thread counts, and reporting format across tools.

## Recommended Datasets

- `simulated`: quick validation and regression checks
- `drosophila`: moderate real-world benchmark with reference-supported evaluation
- `human_chr21`: larger benchmark for runtime and memory scaling

For publication-grade claims, add at least one dataset that matches the intended Raptor use case and one widely recognized public benchmark.

## Metrics To Report

Minimum:

- wall-clock runtime
- peak RSS memory
- contig count
- total assembled bases
- largest contig
- N50

Strongly recommended:

- QUAST metrics
- misassemblies
- NGA50
- genome fraction
- duplication ratio

## Standard Workflow

```bash
# 1. Install tools
./bench/genome_assembly/setup_environment.sh

# 2. Prepare input data
./bench/genome_assembly/download_data.sh simulated
./bench/genome_assembly/download_data.sh drosophila

# 3. Run benchmarks
./bench/genome_assembly/run_benchmark.sh simulated 8
./bench/genome_assembly/run_benchmark.sh drosophila 16

# 4. Aggregate all benchmark runs
python3 ./bench/genome_assembly/summarize_results.py
```

You can also create the comparator environment directly:

```bash
conda env create -f ./bench/genome_assembly/environment.yml
conda activate raptor_bench
```

## Publishing Guidance

Before making performance claims, make sure each reported benchmark includes:

- dataset name and source
- exact command lines
- hardware description
- software versions
- thread count
- result timestamp

Each run generated by `run_benchmark.sh` should be reproducible from the checked-in scripts alone.

## Clean Repo Policy

Keep in git:

- scripts
- benchmark methodology
- summary templates
- curated benchmark reports

Keep out of git:

- raw FASTQ/FASTA downloads
- assembler output directories
- QUAST output trees
- ad hoc local logs

If you want to publish benchmark results in the repo, commit a curated summary document, not the raw output directory.
